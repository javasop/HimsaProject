MADMIN Theme Docs

MADMIN Bootstrap Theme


The function of a great admin UI is to stand out of the way and to let you focus on your data and your controls. This is why we developed MADMIN as a clean, unobstrusive theme inspired by the Metro design philosophy of simplicity and flatness. And we've crafted it on top of the Bootstrap Framework because we know there simply is nothing better to use as solid a foundation for your design.

DOCS

There are just a few simple concepts that you need to grook to make the best use of this theme, so read on to understand them.

Page types

MADMIN gives you a choice of using either:

  • regular pages or
  • panel pages

Regular pages are exactly what you would expect them to be, so there is no point in over-explaining things. This Theme Docs page, for example, and the Login page fall in this category.

Panel pages are divs with CSS class page that are contained in the main page page HTML and are brought in and out of view by means of Javascript. To add a link to a panel, just use an a element like this: <a href="#" data-target-page="page-home">. The data-target-page attribute is simply the id of the .panel element that contains the panel page. All other demo pages besides the ones mentioned above are panel pages.

All .page pages should be children of the #content container and all but the one visible by default should have the class .hidden.

The great advantage of using panel pages is that you pevent page loading delays, making your UI more responsive, and you can have an "admin in one page" if you find this helpful. Or you can load content in them on-demand using AJAX, but you'll have to implement this functionality yourself. Regular pages on the other hand are appropriate if your pages are large and loading them all at once would not be such a good idea. They may also prevent some third party UI components from misbehaving.

The MADMIN side menu

This is one of the highlights of this theme. It is:

  • minimizable — great for netbook screens
  • fixed-responsive — it is fixed if the viewport size is appropriate for viewing its entire content, but stops being fixed if the viewport is too small, allowing regular scrolling
  • capable of on-click or on-hover behavior — you can change the behavior by simply changing the CSS class on it from on-click to on-hover. On click behavior is default and recommended nonetheless, since this is the only mode that is touch-screen friendly. The on-hover mode is by design simpler and stateless since it's supposed to work as a CSS only driven menu (it even functions with Javascript disabled).

Improvement's to Bootstrap's drop-downs

The topbar drop-down menus are simply stiled Bootstrap drop-downs, but they also have behavior enhancements. Add the on-hover CSS class to them and they will pop open just on hovering the mouse over them.

Page-slide in-context help system

This theme integrates the jQuery PageSlide plugin, so you can just add CSS class help-link to any link that you want to open a contextual-help style slide in page.

Easy plotting with Flot

The theme integrates the jQuery Flot plugin, allowing easy graphing, even of real-time data. Checkout Flot's docs to learn more about how to use thin wonderful tool.

Sortable tables

The Tablesorter jQuery plugin is also integrated in the theme, to allow better interaction with tabular data. It is also styled to mathch the overall look of the theme.
